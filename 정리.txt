spa
SPA (Single Page Application)는 웹 애플리케이션의 한 종류로, 전통적인 다중 페이지 웹 애플리케이션과 달리 하나의 HTML 페이지로 시작하고 JavaScript를 사용하여 페이지 갱신을 처리
SPA는 새로운 페이지를 로드하지 않고도 사용자 경험을 개선할 수 있습니다. 
예를 들어, 사용자가 메뉴에서 다른 항목을 선택하면, SPA는 페이지 전체를 새로 고치지 않고 필요한 부분만 업데이트하며 변경된 내용을 표시합니다. 
이것은 일반적으로 더 나은 반응성, 더 빠른 속도 및 더 뛰어난 사용자 경험을 제공합니다.
하지만 SPA가 웹을 앱처럼 사용하는 것은 아닙니다. 
SPA는 웹 기술을 사용하여 구축되며, 대부분의 경우에는 브라우저에서 실행됩니다. 반면에, 네이티브 앱은 모바일 운영체제에 특화된 프로그램으로, 전용 언어나 도구를 사용하여 개발됩니다.

소켓
- 서버와 클라이언트간의 실시간으로 데이터를 주고 받을 수 있는 기술
- 컴퓨터 네트워크를 통해 데이터 통신을 할 수 있는 프로그래밍 인터페이스(API)
- 서버는 소켓을 열어서 클라이언트의 연결 요청을 수락, 클라이언트는 소켓을 열어서 서버에 연결 요청을 보냄
- 소켓을 통해 데이터를 주고받기 위해서는, 두 연결의 소켓이 동일한 프로토콜, IP 주소, 포트 번호 등의 정보를 가지고 있어야 함
- HTTP 통신보다 무겁?기 때문에 동시 접속자의 인원이 크지 않음

HTTP 
- 클라이언트에서 서버쪽으로 요청을 해야 응답을 받을 수 있음
- 클라이언트에서 데이터를 받은 후 연결을 끊음(실시간으로 데이터를 통신할 수 없음)

ORM(Object-Relational Mapping)
- 객체와 관계형 데이터베이스 간의 데이터를 매핑하는 기술
- 객체 지향 프로그래밍 언어에서 사용하는 객체와 속성들을 데이터베이스 테이블과 열(column)에 매핑
- 개발자는 SQL 쿼리를 직접 작성하지 않고, 객체를 통해 데이터베이스에 접근할 수 있음
- 실제 SQL을 사용하는 것 보다 데이터베이스를 다루는데 느림
- 복잡한 서브쿼리같은 것은 ORM으로 표현하기 힘듬
